mysql-日常维护

mysql报错使用mysqlhotcopy 备份报错如下：

DBD::mysql::db do failed: Table './tsoc/cache_clct_dst_ip_5minutely' is marked as crashed and last (automatic?) repair failed at /basoft/tsocba/cupid/database/bin/mysqlhotcopy line 520.

修复方法为

myisamchk -c -r /basoft/tsocba/cupid/database/data/tsoc/cache_clct_dst_ip_5minutely.MYI

修复了下索引，貌似好了

screen

screen支持很多功能，在大数据量交互的情况下使用非常方便。用的最多的特性就是在网络连接中断的情况下screen中的任务还能够保持正常运行状态，现在使用的场景一般为大量数据备份，大量数据拷贝，监控任务的执行等。

screen -S name #创建一个名字叫做name的screen窗口
screen -ls 列出当前用户能够查看到的screen窗口
screen -r name|序列号 通过screen窗口的编号或者名字进入到screen界面
ctrl+a d 退出当前进入的screen窗口
